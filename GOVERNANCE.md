# Governance

Commit-Art is intentionally minimal.

The core principles:

1. Simplicity over complexity
2. Transparency over hidden logic
3. Emergent behavior over enforced design
4. GitHub-native architecture only

Major gameplay changes (canvas size, decay, cooldowns) are decided by the maintainer.

Community suggestions are welcome, but the project will remain focused on its core idea:

Input (Issue) → Mutation (SVG) → Public Record (Commit)
